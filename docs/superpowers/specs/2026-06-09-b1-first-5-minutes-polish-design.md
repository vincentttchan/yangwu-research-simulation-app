# Sprint B1: First 5 Minutes Polish 設計規格

## 目標

把玩家第一次進入遊戲的前五分鐘打磨成一段連續、沉浸、可理解的歷史旅程，使學生能在不依賴老師即時口頭補充的情況下，從登陸頁進入地圖，前往第一個城市，完成第一個證據／事件流程。

## 核心路徑

本 sprint 只聚焦第一條可玩路徑：

`登陸頁 → 開始旅程 → 選李鴻章 → 四拍過場 → 入地圖 → 前往上海 → 第一個證據／事件`

其他人物路線、後期城市、研究 logger、登入、後台資料、完整手機版重排，不屬於本 sprint。

## 設計方向

採用「電影感減法打磨」：

- 少說明，多情境。
- 少教學框，多場景提示。
- 少分類標籤，多玩家自行判斷。
- 少同時可見入口，多清楚下一步。
- 保留歷史厚度，但把第一輪操作負擔降到最低。

## 具體改動範圍

### 1. 登陸頁

- 保留目前黑金、廢墟、晚清危局氛圍。
- 降低非主要入口視覺重量，使「開始旅程」成為唯一主行動。
- 保留 `功業錄`、`人物冊`、`續篇 · 自強之路`，但不可搶走第一眼焦點。
- 避免新增大段功能說明或教學文字。

### 2. 人物選擇頁

- 強化「選擇歷史視角」的感覺，而不是像角色數值設定頁。
- 李鴻章作為首推路線要更明確，但不以硬性教學文字呈現。
- 減少會打斷第一次選擇節奏的輔助資訊。

### 3. 四拍過場

- 保留四拍過場結構。
- 調整節奏為「歷史危局 → 自強起點 → 人物立場 → 入地圖」。
- 過場文字要短、沉、具畫面感。
- 不重複交代玩家任務，因為任務會在地圖與第一城中自然呈現。

### 4. 入地圖第一眼

- 進地圖後第一個目標要一眼可見。
- 上海作為第一站應有明確但不刺眼的視覺召喚。
- 左上提示應短句化，避免像任務清單。
- 可探索城市與目標城市要視覺區分，不能都像同一種紅色警示。

### 5. Coachmark / 提示語氣

- Coachmark 改為「幕僚低語／局勢提示」語氣。
- 避免「①②③ 教你使用介面」的強教學感。
- 第一輪只提示必要行動：前往目標城、查看紅點、完成第一個線索。
- 不展示 DSE、器物／制度／思想等顯性學術分類。

### 6. 第一個城市／事件

- 第一個紅點 hover/tap 的「所見」要短，像場景觀察，不像筆記。
- 第一個證據任務要自然接入，不應像突然進入 worksheet。
- 完成證據後應直接推動玩家進入第一個事件或下一步，不增加重複確認。
- 第一個事件結果要有情緒回聲，但不展開長篇解釋。

## 接受標準

- 新玩家在五分鐘內能完成第一個事件。
- 玩家能自然知道第一步要前往上海。
- 登陸頁、人物選擇、過場、地圖之間沒有重複任務說明。
- 前台不出現 DSE 標籤、顯性分類、worksheet 語氣。
- `功業錄`、`人物冊` 仍可點擊，但不干擾主流程。
- 桌面至少完成一次 smoke test；如工具允許，再檢查 iPad/mobile viewport。

## 驗證

- `npm run check:syntax`
- `npm run check:stability`
- `npm run check:assets`
- `npm run build`
- 手動或自動檢查核心路徑：
  1. 登陸頁主行動清楚。
  2. 可選李鴻章並啟程。
  3. 過場能正常完成。
  4. 入地圖後上海目標清楚。
  5. 第一個城市能進入證據／事件流程。
